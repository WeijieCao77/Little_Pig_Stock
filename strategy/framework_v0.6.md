## 一、策略本质（One-Liner）

在被舍弃催化剂判定的、长期看好的中大盘流动性强股上，
通过三阶段结构（事件前后 run-up + 事件本身价格反应 + 事件后延迟漂移）
分别捕捉机构散户在驱动中的收益。
路径越短，效率越高，胜率越大。

学术对应：
- 阶段 A → Pre-event run-up
- 阶段 B → PEAD（Post-Event Announcement Drift）
- 阶段 C → Delayed sell-the-news drift

## 二、整体投资体系结构

- **Layer 1**：标的池构建（博主报告方法论铺垫）
  - 主题热门行业池筛选
  - 流动性筛查：median 日成交量 ≥1–2 万股
  - 长期价值确认（避免坑）
  - super watch list（30–50 只）
- **Layer 2**：催化剂判定（PEAD 框架）
  - 量化（财报、监管）A/B/C/D
  - 产业链 spillover（待打磨，主题 5）
  - 财务质量打分 + 隐患 + B vs C
- **Layer 3**：交易执行（PEAD 框架）
  - 阶段 A：Pre-event Run-up ✅
  - **决策树：Stage A → B/C ✅ (NEW in v0.6)**
  - 阶段 B：PEAD
  - 阶段 C：Reverse PEAD（L1→L2→L3）
  - 决策日志 + 5 案例 Review + 毕业机制

## 五、阶段 A — Pre-event Run-up（完整规则，继承 v0.4）

### 5.1 时间窗口
- 最早入场：财报前 **7 个交易日**
- 最晚入场：财报前 **2 个交易日**
- 最晚出场：财报当天之前

### 5.2 入场流程（六步）
Gate → 0/1 筛查 → 三档打分 → 主观判断 → 反向证据 → Timing 选择

### 5.3 仓位规则
- 单笔上限：**20% NAV**
- 同时持仓上限：**2 个**

### 5.4 出场规则（三层）
主层 PM 判断 + 兜底层 1 时间 + 兜底层 2 灵活止损

### 5.5 决策日志强制字段
入场 8 个字段 + 出场 4 个字段

### 5.6 Review 机制
每 5 案例回看，<60% 命中率回退到机械纪律

## 六、主题 2 — 财报后决策树（完整规则，v0.6 新增）

### 6.1 决策时间窗口

```
财报公布 → 立即开始观察期
PM 在"信息充分"任何时点决策

硬约束（最晚边界）:
- 盘后财报：第二天美股开盘前
- 盘前财报：当天美股开盘前
```

### 6.2 决策树主流程

**Step 1 — 财报性质判定**

| 路径 | 条件 | 下一步 |
|---|---|---|
| 路径 1 | Beat & Raise（数据强 + 无隐患） | → Step 2A |
| 路径 2 | 温和 Beat / 数据混合 + 有隐患 | → Step 2B |
| 路径 3 | 真正 Miss / 暴雷 | → Step 2C |
| 路径 4 | 难以判定 | → 不做 |

**Step 2A — B 候选：价格反应验证**

| 条件 | 决策 |
|---|---|
| 跳涨 ≥5% | 走 B（进入阶段 B 入场流程） |
| 跳涨 <5% 或其他 | 不做 |

**Step 2B — C 候选：多重证据验证（严格 gate）**

要求**同时**满足以下三项：
1. ✓ 财报前 run-up 过度
2. ✓ 财报前 volume 异常放大
3. ✓ 财报后初步价格反应平淡（±2%）

| 条件 | 决策 |
|---|---|
| 三项都满足 | 走 C（进入阶段 C 入场流程） |
| 任一不满足 | 不做 |

注：Step 2B 的判断维度（run-up 过度 + volume 异常）是借鉴点 2（5 层信号阶段判断）的提前激活。主题 4 时将进一步细化具体阈值。

**Step 2C — 暴雷场景**

| 条件 | 决策 |
|---|---|
| 跳跌 >5% + L1/L2 阶段 | 识别但不做空 |
| 跳跌 >5% + L3 已激活 | 评估做空（具体规则待 L3 主题） |
| 跳跌 <5% | 不做 |

### 6.3 边界情况处理（全部默认保守）

| 边界 | 处理方式 |
|---|---|
| 中间地带价格反应（2–5% 涨） | 不做 |
| 数据复杂（Beat & Miss 混合） | 不做 |
| C 证据部分满足（三项不全） | 不做 |
| 错过决策窗口 | 放弃，等下一个机会 |
| 决策后发现关键信息 | 立即调整，不死扛 |
| 财报时间变更 | 立即按新时间清仓阶段 A |

**核心哲学：框架的硬阈值就是为了过滤模糊机会。"不做"不是策略失败，是策略生效。**

### 6.4 决策日志强制字段
1. 决策时点（精确到分钟）
2. 决策时观察到的信号（财报数据 + 价格反应 + 新闻）
3. 决策结果（B / C / 不做）
4. 边界情况处理（如适用，记录：为什么判定为边界 / 参考了什么因素 / 最终决定 / 决定理由）

### 6.5 边界频率监控
每 5 案例 Review 时统计：
- 决策树各路径的执行情况
- 边界情况占总决策的比例
- 如果边界 >30%，触发框架重新审视（可能阈值有问题）
