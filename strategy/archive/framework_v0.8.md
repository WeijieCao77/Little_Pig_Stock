## 一、策略本质（One-Liner）

在被合格催化剂触发的、长期看好的中大盘流动性资产上，通过三阶段独立结构
（事件前 run-up → 事件后双向跟随）分别捕捉机构和散户行为驱动的中段涨幅或跌幅，
放弃头部和尾部，目标稳定吃中段，而非利益最大化。

学术对应：
- 阶段 A → Pre-event run-up
- 阶段 B → PEAD（Post-Event Announcement Drift）
- 阶段 C → Delayed sell-the-news drift

## 五、阶段 A — Pre-event Run-up（继承 v0.4-v0.7）

- 时间窗口：财报前 7-2 个交易日，最晚财报 1 天清仓
- 入场流程：Gate → 0/1 筛查 → 三档打分 + 主观判断 + 反向证据 + Timing 选择
- 仓位：单笔 ≤20% NAV，同时持仓 ≤2 个
- 出场：PM 主层 + 时间硬约束 + 灵活止损
- 决策日志：入场 8 字段 + 出场 4 字段
- Review：每 5 案例命中率 < 60% 回退机械纪律

## 六、主题 2 — 财报后决策树（继承）

详见 v0.6-v0.7。决策树关键路径：
- Step 1 财报性质判定（Beat & Raise / 温和 Beat + 隐患 / Miss 暴雷 / 模糊）
- Step 2A B 候选：跳涨 ≥5%
- Step 2B C 候选：三项证据齐全（run-up 过度 + volume 异常 + 反应平淡）
- Step 2C 暴雷：跌幅 >5% 且 L1/L2 阶段不做空，L3 评估
- 边界情况全部默认保守 / 放弃 / 改正

## 七、阶段 B — PEAD（继承）

详见 v0.7。关键参数：
- 触发：决策树 Step 2A（Beat & Raise + 跳涨 >5%）
- 决策时间窗口：T+1 开盘 → 决策窗口 → 入场时点
- 入场原则：限价单 + T+1 开盘前任意时间

## 八、🆕 阶段 C — Reverse PEAD（完整规则，v0.8 新增）

### 8.1 触发入口
由主题 2 决策树 Step 2B 触发。

### 8.2 触发条件 Gate（机械，Step 2B 已定）

Gate（机械，Step 2B 已定）：
- 财报性质：温和 Beat + 隐患
- 财报前 5 日涨幅 **>5%**（run-up 过度）
- 5 日 / 30 日 volume 比率 **≥ 2x**（volume 异常）
- 财报后初步价格反应 **≤ ±2%**（平淡）

### PM 主观评估层

#### 必看：5 层信号阶段判断 + 理由 + 反对意见
- 标的处于 ①②③④⑤ 哪一层？
- 核心理由是什么
- 反对意见是什么（如有大不同意，应该重新设计）

#### 选看：模拟候选清单（4 个备选）
1. Insider Selling
2. 期权 IV / Implied Move
3. 散户讨论度 vs 产业链谋不对称
4. 大盘环境（VIX / risk-on/off）

**PM 综合判断：走 / 不做 / 仓位多少**

### 8.3 决策时间线（Step 2B 触发后）

```
[T0]      财报公布 + Gate 通过
   ↓
[T0+T+1 月盘前]  PM 评估 + 决策 + 推理结
   ↓
[T+1 开盘瞬间]   决策最晚边界 ⏰ 限价单触发执行
```

### 8.4 L1/L2/L3 渐进激活

| 层级 | 行动 | 工具 | 仓位 | 毕业条件 |
|---|---|---|---|---|
| **L1** | 仅识别，日志记录 | 无 | **0%** | 5 次识别 + 60% 命中率 → L2 |
| **L2** | 间接反向暴露 | 反向 ETF（**SQQQ**） | ≤ **10% NAV** | 3 次 + 50% 命中率 → L3 |
| **L3** | 直接做空 | **Put 期权**（ATM + 近月 1-3 周） | 验证期内 ≤ **2% NAV**，后 ≤ 8% NAV | 持续监控 |

**互斥约束**：L2 和 L3 不能同时持有（简化反向暴露管理）

**L3 账户规模注意**：当前账户 **~$7.1K**（2026-05-28 轧差估算；此前写的 ~$8.5K 偏高、未经核实），2% = ~$140，单张 Put 通常 $200-$500 → **以当前规模,单张近月 Put 就已超 2% 验证期上限**，激活 L3 前必须重新评估账户规模匹配度（很可能要等账户更大或改用价差结构）。

### 8.5 入场方式

- 所有有交易的层级（L2/L3）→ 限价单
- 挂单时点：T+1 开盘前任意时间
- 与阶段 B 一致的执行节奏

### 8.6 仓位规则

```
与事件驱动池的关系
A + B + C 共享同一个"事件驱动池"
- 同时持仓上限：2 个（任意阶段组合）
- 总账面事件暴露：≤ 40% NAV

🚫 互斥约束
L2 和 L3 不能同时持有
```

### 8.7 出场规则（阶段 C 专属）

**主层：PM 主观判断** 🟢
- 看到散户撤资结束 / 反弹启动 / 达到目标主动出
- 日志记录"为什么这天出"

**兜底层 1：时间硬约束** 🟠
- 持有满 7 个交易日强制清仓
- 或接近 T+5 次估值开始 2 天强制减仓

**兜底层 2：反向止损（机械严）**
- 从入场价 **5%** 自动止损（机械）
- L2/L3 没激活时反向 ETF 保护或 FOMO 触发执行
- ❌ 不放分批止盈 —— 阶段 C 持有期短，FOMO 倍量化执行

### 8.8 决策日志强制字段

**入场记录**：
1. Gate 检查情况（Step 2B 三项证据）
2. 5 层信号判断（阶段 + 理由 + 反对意见）
3. PM 数学评估 + 候选其他信号清单
4. 综合判断结论
5. 仓位决定 + 理由
6. 入场时点（限价价格 + 工具选择）

**出场记录**：
1. 出场触发层级（主层 / 兜底 1 / 兜底 2）
2. 出场具体理由
3. 实际入场价 / 出场价
4. 持有期，收益率
